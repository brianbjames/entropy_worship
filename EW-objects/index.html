SYSTEM
NETWORK MAP
Live topology visualization of the distributed patch network. Shows all active rooms, their connections, and signal flow hierarchy as a graph. Nodes represent objects currently online, and edges represent active room subscriptions between them. Supports public and private dashboard rooms — private dashboards can monitor private room URLs added manually. Use it to see what is currently connected on the network, trace signal paths, and verify your patch routing at a glance — especially useful when multiple users are collaborating across machines.

// COLLABORATIVE DISTRIBUTED MODULAR RELAY NETWORK SYNTHESIZER //

Welcome to an experimental modular audio patching environment that runs entirely in the browser as a distributed and collaborative audio network.

Each object is a standalone signal-processing node which handles synthesis, processing, and routing at the module level. No other modular synthesis environment uses the network as its native patching medium — turning WebSocket room URLs into patch cables, and browsers into rack modules.

Control signals flow between them at ~30Hz — the same publish/subscribe pattern that Max/MSP uses with send/receive, SuperCollider uses with buses, or Pd uses with send/receive, except in this case the connection layer is the network itself. Modules can live in tabs on one machine, across machines on a LAN, or across the internet with multiple players patching into the same signal graph simultaneously.

// HOW TO CONNECT //

1. Open any object below — start with the OSCILLATOR. On load, it handshakes with the WebSocket server and claims a unique room name (something like osc-A2WG9E). That room name is your output bus. You will see it in the header bar next to the status indicator.

2. Open a second object in a new tab — a FILTER, a DELAY, anything that accepts input. At the bottom of the page you will find labeled input ports, each with a text field and a LINK button. These are your inlets.

3. Go back to the source module, click COPY URL to grab its room name, then paste that into the destination module's input field and click LINK. The status dot goes green — you have a live connection. Think of it as plugging a patch cable from one module's output jack into another's input jack.

4. Parameter changes on the source module now propagate in real-time to every subscriber. Sweep the oscillator's frequency and the downstream filter tracks it. Chain as many modules as you want — the signal graph is arbitrary and there is no fixed limit on connection depth or fan-out.

5. To bring someone else into the patch, send them a room URL. They paste it into their own module on their own machine and they are subscribed to your signal output. Multiple users can monitor or process the same room simultaneously — same as sharing a Bus in SuperCollider, except the Bus spans the network.

6. Rooms can be set to private, restricting signal flow to only those who know the room name — useful for isolating sub-patches or running a closed session. The server itself is a standard Node.js WebSocket process — clone the repo, run npm install and npm start inside the EW-objects/ directory, and you have your own signal relay running on localhost:3001. Point your modules at that local address instead of the public server and all traffic stays on your machine or LAN — no internet round-trip, minimal latency, and only devices you allow on your network can connect. This is ideal for live performance setups, private jam sessions, or development without depending on an external host.
