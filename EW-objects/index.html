// COLLABORATIVE DISTRIBUTED MODULAR RELAY NETWORK SYNTHESIZER //

EWObjects is a novel modular audio synthesis environment that runs entirely in the browser as a distributed network. Each object is a self-contained module — an oscillator, a filter, a delay, a sequencer — that generates, processes, or routes audio and control signals.

What makes this different from other browser synths: every module connects to a shared WebSocket server, and room URLs act as patch cables. Open an oscillator in one tab and a filter in another. Copy the oscillator's room URL, paste it into the filter's audio input, click LINK — and audio flows between them in real-time. These tabs can be on the same machine, on different machines across a network, or shared between multiple users collaborating on the same patch simultaneously.

// HOW TO CONNECT //

1. Open any object above — say, the OSCILLATOR. When the page loads it automatically connects to the WebSocket server and generates a unique room name (something like osc-A2WG9E). You'll see it in the header bar next to the status dot.

2. Open a second object in another tab — a FILTER, a DELAY, whatever you want to route into. Look for the input fields at the bottom of the page. Each input port has a text field and a LINK button.

3. Go back to the first tab, click COPY URL to grab its room URL, then paste that into the second tab's input port and click LINK. The status dot turns green — the connection is live.

4. Every knob you turn on the source module is now mirrored in real-time on the receiving module. Change the oscillator's frequency and the filter follows. Chain as many modules together as you want — there's no limit.

5. To jam with someone else, just send them a room URL. They paste it into their own module on their own machine and they're patched into your signal chain. Multiple users can subscribe to the same room simultaneously.
